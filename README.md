# Soul Moments 灵魂说说

> 让Bot 像真人一样发QQ说说、阅读用户说说、评论、互动等，内容完全由角色人格驱动

## 功能

- 🎭 **角色驱动** - 说说内容完全由角色人格决定，不同角色写出完全不同风格的动态
- 📅 **每日随机** - 每天自动规划发 0-N 条，发布时间随机分布在活跃时段内
- 💰 **极致省 token** - 只在真正发布时调一次 LLM，无额外判断请求
- 🤖 **多 Bot 支持** - 一个 AstrBot 上多个机器人？逗号隔开就行，各自独立状态
- 🔌 **完全独立** - 不依赖其他插件，即装即用

## 工作原理

1. 每天凌晨（或首次启动时）自动制定当天计划：随机决定发几条、几点发
2. 到了计划时间，调用 LLM 以角色身份生成一条说说
3. 通过 QZone API 自动发布

> 💡 举例：配置 `0-2` 条/天，08:00-23:00 活跃  
> → 今天抽到发 1 条，时间 14:37  
> → 到 14:37 自动生成 + 发布，全天只花 1 次 LLM 调用

## 配置说明

所有配置通过 AstrBot WebUI 设置：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| platform_id | 平台 ID，多个用逗号隔开 | (必填) |
| daily_range | 每天发布条数区间 | 0-2 |
| active_hours | 活跃时段 | 08:00-23:00 |
| timezone | 时区 | Asia/Shanghai |
| max_length | 说说最大字数 | 200 |
| provider_id | 使用的模型（可选） | (默认模型) |
| prompt_override | 自定义提示词（高级） | (留空) |

### daily_range 格式

- `0-2`：每天随机 0~2 条（可能不发）
- `1-3`：每天至少 1 条，最多 3 条
- `0-0`：永远不发（相当于关闭）
- `2-2`：每天固定 2 条

## 用户命令

| 命令 | 说明 |
|------|------|
| /说说 状态 | 查看今日计划、发布记录 |
| /说说 发布 | 手动触发立即发布一条 |

命令别名：`/moments`、`/ss`

## 技术原理

- 通过 NapCat 的 `get_cookies` 获取 QZone 登录态
- 计算 `g_tk` (CSRF token) 后，HTTP POST 到 QZone 发说说 API
- 说说发布在 QQ 空间说说/动态中可见

## 要求

- AstrBot >= 4.18.0
- NapCat（OneBot v11 适配器）
- NapCat 需与 AstrBot 通过 WebSocket 连接

## 安装

将插件目录放入 AstrBot 的 `data/plugins/` 目录下，重启 AstrBot 即可。

## 注意事项

- QZone API 为非官方接口，可能因腾讯调整而失效
- 发布失败时会自动跳过（不会反复重试浪费 token）
- 建议 `daily_range` 不要设太大，1-3 条/天比较自然
